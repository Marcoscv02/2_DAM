# **1. Concepto del EntityManager**üßô‚Äç‚ôÇÔ∏è



üß© El _**EntityManager**_ es el n√∫cleo del mecanismo de persistencia de Java Persistence API (JPA). Este gestor es el encargado de **controlar el ciclo de vida de las entidades** y **realiza operaciones** como la persistencia, consulta, eliminaci√≥n y actualizaci√≥n de datos.



### ‚ú®**Funciones Principales del EntityManager:**

- **Persist**: Permite almacenar una entidad en la base de datos.
- **Merge**: Actualiza una entidad persistente con los cambios realizados en su instancia.
- **Remove**: Elimina una entidad persistente de la base de datos.
- **Find**: Recupera una entidad de la base de datos basada en su clave primaria.
- **CreateQuery**: Realiza consultas JPQL (Java Persistence Query Language).

### üê≤**Persistence Context**:

- Es un contexto dentro del EntityManager que **mantiene las entidades que han sido cargadas o creadas**. Una entidad se considera persistente cuando est√° dentro de este contexto y ha sido asociada al EntityManager.

### üéÜ**Persistencia y entidades**:

<aside> üéóÔ∏è

Una entidad solo se convierte en persistente cuando se invoca el m√©todo `persist()` en el EntityManager.

</aside>

- Solo una instancia de Java con una misma identidad persistente puede existir en un contexto de persistencia en cualquier momento.

# **3. Operaciones CRUD**

JPA permite realizar operaciones b√°sicas como **Crear**, **Leer**, **Actualizar** y **Eliminar**. Estas operaciones se efect√∫an en el contexto de un EntityManager.

## **3.1. Persistir una Entidad**

- <span style="color:rgb(255, 0, 0)"><u>Persist</u></span>: Consiste en almacenar una entidad en la base de datos.
    
- Un **Empleado** se crea como una instancia transitoria que, al ser pasada a `persist()`, se convierte en persistente.
    
    ```java
    Empleado emp = new Empleado(158); // Instancia de la entidad Empleado
    em.persist(emp);  // Persistir en la base de datos
    ```
    
- **Ejemplo completo**:
    
    ```java
    public Empleado createEmpleado(int id, String nome, long salario) {
        Empleado emp = new Empleado(id);
        emp.setNome(nome);
        emp.setSalario(salario);
        em.persist(emp);
        return emp;
    }
    ```
    
- **Excepciones**:
    
    - Si ocurre un error durante la persistencia, se lanza una `PersistenceException`.

## **3.2. Obtenci√≥n de una Entidad**

- <span style="color:rgb(255, 0, 0)"><u>Find</u></span>: Recupera una entidad de la base de datos. Utiliza la clase entidad y la clave primaria.
    
    ```java
    Empleado emp = em.find(Empleado.class, 158);
    ```
    
- **Ejemplo**:
    
    ```java
    public Empleado findEmpleado(int id) {
        return em.find(Empleado.class, id);
    }
    ```
    
- **Comprobaci√≥n de null**:
    
    - Si la entidad no se encuentra en la base de datos, `find()` devuelve `null`.
    
    ```java
    if (emp != null) {
        System.out.println("Empleado encontrado: " + emp);
    } else {
        System.out.println("Empleado no encontrado.");
    }
    ```
    

## **3.3. Eliminaci√≥n de una Entidad**

- **<span style="color:rgb(255, 0, 0)"><u>Remove</u></span>: Elimina una entidad persistente de la base de datos.
    
- La entidad debe estar previamente gestionada (persistida dentro del contexto).
    
    ```java
    Empleado emp = em.find(Empleado.class, 158);
    em.remove(emp);  // Eliminar la entidad
    ```
    
- **Ejemplo**:
    
    ```java
    public void removeEmpleado(int id) {
        Empleado emp = em.find(Empleado.class, id);
        if (emp != null) {
            em.remove(emp);
            System.out.println("Empleado eliminado.");
        }
    }
    ```
    

## **3.4. Actualizaci√≥n de una Entidad**

- **<span style="color:rgb(255, 0, 0)"><u>Merge</u></span>: Actualiza una entidad persistente con los cambios hechos en su instancia.
    
- **setSalario**: Cambiar el salario del empleado.
    
    ```java
    Empleado emp = em.find(Empleado.class, 158);
    emp.setSalario(1000000);  // Cambiamos el salario del empleado
    em.merge(emp);  // Persistimos el cambio
    ```
    
- **Ejemplo**:
    
    ```java
    public Empleado raiseSalarioEmpleado(int id, long cantidad) {
        Empleado emp = em.find(Empleado.class, id);
        if (emp != null) {
            emp.setSalario(emp.getSalario() + cantidad);
            em.merge(emp);  // Actualizaci√≥n persistente
        }
        return emp;
    }
    ```
    

## **3.5. Ejemplo Completo de CRUD**

```java

        EntityManager em = emf.createEntityManager();

        // Obtener el empleado creado

        em.merge(emp);
        em.getTransaction().commit();

        // Eliminar el empleado

        em.remove(emp);
        em.getTransaction().commit();

        emf.close();
    }

    public static Empleado createEmpleado(int id, String nome, long salario) {
        EntityManager em = emf.createEntityManager();
        Empleado emp = new Empleado(id);
        emp.setNome(nome);
        emp.setSalario(salario);
        em.persist(emp);
        em.close();
        return emp;
    }
}
```

# **4. Transacciones** üîÑ



>‚ö†Ô∏è Las operaciones de persistencia deben ser envueltas en **transacciones** para asegurar que los cambios se mantengan o se reviertan correctamente.



- Para manejar las transacciones en JPA, utilizamos la clase `EntityTransaction`.
    
    ```java

    ```
    
- Se confirma la transacci√≥n mediante `commit()`.
    
    ```java

    ```
    
- **Ejemplo completo**:
    
    ```java

    createEmpleado(158, "John Doe", 45000);
    em.getTransaction().commit();
    ```

## **5.1. Introducci√≥n a las Consultas en JPA**



>üé≠ En **JPA (Java Persistence API)**, una **consulta** es una acci√≥n que permite acceder a datos de la base de datos mediante el uso de **entidades**. Las consultas pueden ser construidas de manera **din√°mica** o bien definidas de forma **est√°tica** (con nombre).



- üåü **Consultas Din√°micas**: Se construyen en tiempo de ejecuci√≥n como cadenas de consulta JPQL.
- üîß **Consultas con Nombre**: Se definen en tiempo de compilaci√≥n utilizando el archivo de metadatos de la entidad o un archivo de consulta.

**Cada consulta se realiza utilizando el EntityManager** que act√∫a como la "f√°brica de consultas", encarg√°ndose de manejar las operaciones sobre las entidades.

## **5.2. Tipos de Consultas**

### **5.2.1. Consultas Din√°micas**

> Las **consultas din√°micas** se construyen utilizando cadenas de consulta **JPQL (Java Persistence Query Language)** durante la ejecuci√≥n.

- üíº **M√©todo `createQuery()`**: Este m√©todo en el **EntityManager** crea la consulta a partir de una cadena de JPQL.

### **5.2.2. Consultas con Nombre (Est√°ticas)**

## **5.3. Ejecuci√≥n de Consultas**

Las consultas se pueden ejecutar de diversas maneras seg√∫n el tipo y la necesidad. Existen dos maneras principales:

### **5.3.1. `getResultList()`**

> ‚ö†Ô∏è Posibles e**xcepciones**:
